# sheets_export — общая Google-таблица открытых тикетов/заявок

Пишет (не читает) в Google Таблицы: одна общая таблица, куда сводятся **открытые**
тикеты Fibbee ERP (`state='open'`) и **незакрытые** заявки Service Desk
(`status != 'Закрыта'`) — со ссылками на источник, единый вид для всех, кому
нужно посмотреть, что сейчас в работе, без входа в каждую систему отдельно.

История закрытых тикетов сюда не попадает — Fibbee сама по себе накопила 3700+
тикетов, для общей таблицы "что сейчас в работе" это не нужно; сводные отчёты
по всей истории — отдельно, см. `core/reporting/`.

## Настройка Google Cloud

Тот же проект и тот же OAuth-клиент, что у gmail/drive (`sheets-495712`):

1. Google Sheets API в этом проекте уже включён (использовался для чтения таблиц
   в `connectors/drive`) — включать заново не нужно.
2. На вкладке **Data Access** (Google Auth Platform) добавь scope `spreadsheets`
   (полный, НЕ `.readonly` — этому коннектору нужна запись).
3. `credentials.json` уже скопирован в эту папку (тот же клиент, что у gmail/drive).

## Запуск

```
cd connectors\sheets_export
python -m venv venv
venv\Scripts\pip install -r requirements.txt
copy .env.example .env
venv\Scripts\python sync.py
```

Первый запуск — браузер, вход, согласие на запись в Google Таблицы (отдельное
окно от Drive/Gmail — другой scope, свой `data/token.json`). Тем же прогоном
создаётся сама таблица (Sheets API это позволяет без Drive-прав) — в консоли
напечатается ссылка на неё и `sheet_state.json` рядом запомнит её ID.

**Обязательный ручной шаг после первого запуска:** открой ссылку на таблицу и
нажми "Поделиться" — дай доступ нужным людям или всему домену. У коннектора
нет Drive-scope, программно раздать права он не может.

Дальше — просто `python sync.py` периодически (см. ниже про планировщик).

## Что в таблице

Одна вкладка "Тикеты": строка "Обновлено: ..." сверху, дальше заголовки и
строки — Источник / № / Статус / Заголовок / Создано / Ссылка / Описание.
Столбец "Ссылка" — обычный URL как текст; Google Таблицы сами превращают его
в кликабельную ссылку при вставке через API (то же самое, что при ручном
вводе в интерфейсе).

**Таблица целиком перезаписывается на каждом запуске** — вручную в ней
редактировать что-либо бессмысленно, следующий прогон затрёт. Так проще и
надёжнее, чем построчный upsert: закрытые с прошлого раза тикеты гарантированно
пропадают из таблицы, а не остаются висеть до отдельной логики удаления.

## Автообновление

Чтобы таблица обновлялась сама (как Tracker/Service Desk/Fibbee) — добавь
`sheets_export` в `FREQUENT_JOBS` в `core/scheduler/run.py` (это уже сделано,
если ты не убирал строку) — раз в `SYNC_INTERVAL_HOURS` часов (по умолчанию 2).

## Точность ссылок

Ссылки строятся по `core/reporting/link_templates.yaml` — шаблоны для
Service Desk и Fibbee там помечены как непроверенные (не сверялись с реальным
адресом конкретного тикета в браузере). Открой пару строк из готовой таблицы
и сверь, что ссылка ведёт именно на тот тикет — если нет, поправь шаблон в
`link_templates.yaml`, это единственное место, где он хранится (используется
и здесь, и в сводных отчётах).
