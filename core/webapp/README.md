# webapp — веб-интерфейс (тёмная тема)

FastAPI + HTMX + Jinja2, рендер на сервере, без сборки фронтенда. Работает
локально на твоём ноутбуке (`127.0.0.1`), никуда не публикуется. Покрывает
все модули из `functional_plan_ui.md`: задачи, сводные отчёты, база знаний,
Discord + мониторинг.

## Запуск (Windows) — самый простой способ

Двойной клик по **`start.bat`** (или по `start_webapp.bat` в корне проекта) —
при первом запуске сам создаст venv, поставит зависимости и откроет браузер.
Второй и последующие разы — сразу стартует, без переустановки.

Порт — **9000** (не 8000, он занят другими приложениями): **http://127.0.0.1:9000**

## Установка и запуск вручную (если нужен контроль/отладка)

```
cd core\webapp
python -m venv venv
venv\Scripts\pip install -r requirements.txt
venv\Scripts\python -m uvicorn main:app --reload --port 9000
```

Свой venv (не общий с `core/router_ai`) — здесь дополнительно fastapi/uvicorn/numpy.
Ключ OpenRouter не дублируется — читается напрямую из `core/router_ai/.env`.

## Разделы

- **Задачи** — список заявок/тикетов/задач, группировка оперативные (Service
  Desk + Fibbee ERP) / стратегические (Tracker), недавно обновлённые.
  Группировка по темам внутри источника — ещё не сделана (см.
  `functional_plan_ui.md` §1.1, следующий шаг).
- **Отчёты** — то же, что `core/reporting/generate.py`, но через форму в
  браузере: ответ инлайн со ссылками на источники, либо скачать HTML.
- **База знаний** — чат по Google Drive + Discord. Сначала нужно построить
  индекс (не строится сам при старте сервера, это отдельная команда — токены
  тратятся только явно, по твоей команде):
  ```
  venv\Scripts\python kb_index.py build
  ```
  Индекс кэшируется в `data/kb_index.db` (гитигнорится) — повторный запуск
  пересчитывает эмбеддинги только для новых/изменившихся чанков. По
  умолчанию Discord индексируется за последние 90 дней (`--discord-days`),
  Drive — весь текст целиком.
- **Discord + мониторинг** — хронология с подсветкой упоминаний
  monitoring/service team (см. `monitoring_config.yaml` — сейчас fallback по
  ключевым словам, ID ролей ещё не настроены), обобщение за период,
  статус комплексов (последняя синхронизация Fibbee), тикеты ERP + рабочая
  ссылка на нативный дашборд комплекса.

## Известные упрощения v1 (см. functional_plan_ui.md)

- "Обновления" в задачах — сортировка по `updated_at`/`closed_at`, а не
  полноценный diff-слой (`task_updates`) — тот появится с общим read-layer.
- Группировка задач по темам — не сделана, только по источнику.
- База знаний — брутфорс косинус в numpy по SQLite-кэшу, не pgvector (осознанно,
  см. докстринг `kb_index.py` — на этом объёме данных избыточно).
- Подсветка упоминаний в Discord — по ключевым словам, не по ID ролей (не
  переданы пользователем).
- Статус комплексов — по последней синхронизации `sync.py`, не живой
  поллинг (нужен cron/планировщик — не настроен).
