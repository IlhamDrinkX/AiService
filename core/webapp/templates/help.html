Помощь

Что делает приложение и что нужно сделать при первом запуске.

Что это

Локальный помощник службы поддержки — собирает данные из Discord, Google Drive, Tracker, Service Desk и Fibbee ERP в одном месте и даёт задавать вопросы по ним на естественном языке. Работает целиком на этом компьютере — данные никуда, кроме выбранного провайдера модели (OpenRouter или корпоративный Gemini), не уходят.

Задачи

Единый список заявок/тикетов/задач из Service Desk, Fibbee ERP и Tracker. Фильтр по любому полю (можно выбрать несколько значений сразу), сортировка по клику на заголовок столбца, поиск по всем полям разом. Ссылка "перейти" в каждой строке ведёт на оригинал в исходной системе.

Отчёты

Задай вопрос по заявкам/тикетам/задачам/сообщениям Discord за выбранный период — модель ответит текстом со ссылками на конкретные источники, на основе которых сделаны выводы (номера в квадратных скобках — кликабельные ссылки). Если явно попросить график или напрашивается числовая разбивка (например "посчитай заявки по статусам") — модель может добавить диаграмму прямо в ответ. Кнопка "Скачать HTML" — тот же отчёт файлом, можно переслать.

Уровень модели: nano — самый дешёвый и быстрый (короткие вопросы), flash — обычный уровень по умолчанию, pro — для сложных вопросов сразу по многим источникам (Claude через OpenRouter, платно). Провайдер: корпоративный Gemini бесплатный, но иногда упирается в дневной лимит бесплатного тира — тогда переключись на OpenRouter (платно, но без лимитов).

База знаний

Чат-бот по всем документам Google Drive и истории Discord — вопрос на естественном языке, ответ со ссылками на конкретные документы/сообщения. Индекс обновляется командой python kb_index.py build (см. README в core/webapp/) — тратит токены только на изменившиеся с прошлого раза данные, не пересчитывает всё заново.

Discord + мониторинг

Хронология сообщений с подсветкой упоминаний monitoring/service team, обобщение за произвольный период (можно уточнить тему, например "проблемы с молочкой"), таблица комплексов по данным Fibbee (с фильтром и ссылкой на нативный дашборд), тикеты Fibbee за последнюю неделю.

Токены

Баланс и расход по ключу OpenRouter, статистика по провайдеру/задаче/модели, таблица доступности всех источников (с кнопкой "Проверить сейчас" — не ждать фоновый цикл), и кнопки управления: принудительная синхронизация Google Drive вне расписания, перезапуск и полная остановка веб-морды.

Первый запуск — авторизация

При первом запуске каждый источник данных нужно один раз авторизовать — дальше токен сохраняется локально и повторного входа не требует (пока не истечёт/не будет отозван вручную):

Google (Gmail, Drive, общая Google-таблица) — при первом запуске sync.py соответствующего коннектора откроется браузер, нужно войти под своим Google-аккаунтом и подтвердить доступ. Если стоит VPN — он может заблокировать саму страницу входа Google (accounts.google.com); в этом случае временно отключи VPN на время входа, затем включи обратно.

Discord — нужен токен бота (создаётся в discord.developer.com, см. SETUP_NEW_USER.md в корне проекта) — один раз вписывается в connectors/discord/.env, дальше бот подключается сам при каждом запуске.

Tracker / Service Desk / Fibbee ERP — логин/пароль или токен один раз вписывается в .env соответствующего коннектора (см. README каждого коннектора) — веб-морда сама переиспользует эти же данные при синхронизации.

OpenRouter / корпоративный Gemini — ключи API один раз вписываются в core/router_ai/.env (OPENROUTER_API_KEY, GEMINI_API_KEY).

Полная пошаговая инструкция для нового пользователя (свой Discord-бот, свой Google OAuth-клиент и т.д.) — в SETUP_NEW_USER.md в корне проекта.
